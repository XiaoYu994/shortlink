# Phase 7 回滚操作手册

## 1. 触发条件

- 关键接口 5xx 比例持续超阈值
- 跳转链路延迟持续异常
- MQ 消费失败持续增长
- 发布后核心功能不可用

## 2. 回滚原则

1. 先止血：回滚出现问题的业务服务
2. 后定位：保留监控栈用于问题分析
3. 最小影响：避免全量回滚，优先单服务回退

## 3. 业务服务回滚步骤

1. 定位异常服务版本
2. 将该服务切回上一稳定版本
3. 验证 `/actuator/health` 与核心 API
4. 观察 15 分钟关键指标

## 4. 监控栈回滚步骤

1. 若监控栈本身异常，可停止以下容器：
   - `shortlink-prometheus`
   - `shortlink-grafana`
   - `shortlink-alertmanager`
2. 保持业务服务继续运行
3. 恢复上一版监控配置后再重启监控容器

## 5. 回滚验证

- [ ] 核心链路恢复正常
- [ ] 告警级别恢复到可接受范围
- [ ] 无新增数据一致性问题

## 6. 事后记录

- 回滚开始/结束时间
- 影响范围
- 根因初判
- 后续修复项
